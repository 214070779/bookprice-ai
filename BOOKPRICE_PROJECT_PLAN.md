## 1. 执行摘要

**一句话:** 给二手书卖家的 AI 定价+选品助手。

**痛点:** 40 万孔夫子卖家目前靠人工翻排行榜选品、凭感觉定价。滞销率高达 70%，大量利润浪费在错误定价和错误选品上。

**解决方案:** AI Agent（MCP Server）+ Web 面板。输入 ISBN → 3 秒内返回全网价格、竞争分析、建议定价、周转预测。

**独特价值:** 市面上有采集工具（把书搬来搬去）、有调价工具（针对淘宝京东），但**没有专为二手书设计的 AI 定价+选品引擎**。这是空白。

**商业模式:** Freemium。免费额度引流，Pro 订阅变现。
- 免费: 50 次 ISBN 分析/月
- Pro: ¥29/月（个人），¥99/月（店铺版，多账号）

**目标:** 6 个月内获取 1000 付费用户，月经常性收入（MRR）¥50,000+。

## 2. 市场背景与机会

### 2.1 市场规模

| 指标 | 数据 |
|------|------|
| 孔夫子旧书网年交易额 | 12 亿元 |
| 孔夫子卖家数 | 40 万（2 万家书店 + 45 万家书摊） |
| 孔夫子买家数 | 2100 万 |
| 30 岁以下卖家占比 | 30%（约 12 万年轻人） |
| 中国旧书回收市场规模 | 58.7 亿元（2025），CAGR 15.9% |
| 普通二手书毛利率 | 30-40%（且持续下降） |
| 卖家平均滞销率 | 70% |

### 2.2 竞争格局

| 竞品类型 | 代表 | 做什么 | 不做啥 |
|----------|------|--------|--------|
| 上货工具 | 黑谍软件、展翅软件 | ISBN 采集 + 批量上架 | 不分析价格、不推荐选品 |
| 电商调价 | 哪吒调价 | 淘宝/京东 Buy Box 自动调价 | 不做二手书、不做选品 |
| ISBN 数据 | 极速数据 | 返回图书基本信息 | 不做价格分析、不做趋势 |
| C2B2C 平台 | 多抓鱼 | 自营二手书买卖 | 不对外提供定价工具 |
| 综合平台 | 闲鱼、转转 | C2C/C2B2C 交易 | 不提供卖家辅助工具 |

### 2.3 市场空白

**核心发现:** 二手书领域存在一个"工具断层"。

卖家需要的不是"把书搬上架"（已有工具），也不是"在淘宝抢购物车"（哪吒的场景），而是：

> **"这本书我该卖多少钱？该不该收这本书？"**

这恰恰是 AI 最擅长的事情——而目前市场上没有人做。

### 2.4 为什么现在切入

1. **孔夫子开放平台已上线**（open.kongfz.com）→ 可以合法获取商品数据
2. **AI 成本大幅下降** → 调用大模型做定价分析成本已低于 0.01 元/次
3. **二手书利润率持续走低** → 卖家对精细化运营工具的需求愈发迫切
4. **年轻卖家比例上升** → 30 岁以下占比已达 30%，对工具接受度高
5. **你已有完整的 MCP 开发+上架+变现经验** → 复用 code-scanner 的流程

## 3. 目标用户

### 3.1 核心用户画像

| 画像 | 描述 | 规模 | 付费意愿 |
|------|------|------|---------|
| **孔夫子年轻卖家** | 30 岁以下，在孔夫子开书摊，月营收 5K-30K | ~12 万 | 中高 |
| **闲鱼二手书商家** | 在闲鱼批量卖书，有出版物经营许可证 | ~5 万 | 中 |
| **二手书回收商** | 从线下回收书籍，需要快速估价 | ~1 万 | 高 |
| **多平台卖家** | 同时在孔夫子+闲鱼+淘宝卖书 | ~3 万 | 高 |

### 3.2 用户需求优先级

1. **定价**（最痛）: "这本书卖多少钱合适？卖高了卖不动，卖低了亏本"
2. **选品**（次痛）: "收什么书能快速卖掉？什么书现在有需求？"
3. **库存优化**（第三）: "仓库里积压的书该降价多少处理？"
4. **跨平台套利**（进阶）: "哪本书在 A 平台低价、B 平台高价？"

## 4. 产品设计

### 4.1 产品形态

**双模式交付:**

**模式 A: MCP Server（AI Agent 原生）**
- 5 个 MCP 工具，可接入 Claude/Cursor 等 AI 助手
- 用户直接对 AI 说"帮我看看这本书"→ AI 调用工具返回分析
- 适合懂技术的卖家，以及未来 AI 原生工作流

**模式 B: Web 面板（主流用户）**
- 微信小程序/Web 端，扫码即可用
- 适合 95% 的非技术卖家
- 提供批量操作、数据看板、库存管理

1. **定价中心**

   - 定价效果追踪

## 6. AI 引擎设计

### 6.1 定价算法

**三层定价模型:**

```
定价 = f(市场基准价, 供需因子, 品相因子, 时效因子)
```

- **市场基准价:** 孔夫子/闲鱼同品相中位价（实时抓取）
- **供需因子:** 在售数量 ÷ 月销量 → 供过于求则降价
- **品相因子:** 全新/九五品/八五品/七五品 → 乘数调整
- **时效因子:** 滞销天数 → 自动降价建议，60 天未售 → 建议降至成本价

**定价策略输出:**

```
建议售价 = 成本价 × (1 + 目标毛利率)
if 建议售价 > 市场中位价: 下调至中位价
if 市场中位价 - 成本价 < 10元: 标记为"不推荐收购"
```

### 6.2 选品推荐算法

**评分模型:**

```
选品分 = w1 × 利润率 + w2 × 周转速度 + w3 × 需求稳定性 - w4 × 竞争强度
```

- **利润率:** (预期售价 - 成本) / 成本
- **周转速度:** 近 30 天销量 / 在售数
- **需求稳定性:** 月度销量波动率（低波动 = 高稳定性）
- **竞争强度:** 在售商家数

**Top N 推荐:** 按选品分排序，过滤掉利润率 < 30% 或周转 > 60 天的书。

### 6.3 数据积累飞轮

这是整个产品的核心护城河:

```
用户越多 → ISBN 定价请求越多 → 积累更多成交数据
→ AI 模型越精准 → 用户更依赖工具 → 用户越多
```

**冷启动策略:**
1. 初始用孔夫子开放平台的历史数据训练基线模型
2. 用 ISBN 数据库的基本信息 + 简单的规则引擎（不需要大量数据）
3. 用户使用后积累真实定价-成交数据，持续优化

- 仅采集公开可访问的价格/销量信息
- 遵守 robots.txt 和平台使用条款
- ISBN 查询结果缓存 24 小时（降低成本 + 减少平台压力）

### 8.3 冷启动数据策略

**第一阶段（MVP）：**
- ISBN API 获取图书基础信息
- 从孔夫子开放平台获取在售数据
- 用规则引擎定价（不依赖历史成交数据）
- 规则: "市场中位价 ± 供需调整"

**第二阶段（增长）：**
- 用户定价-成交数据积累
- 开始训练 ML 模型
- 从"规则定价"过渡到"AI 定价"

**第三阶段（护城河）：**
- 独有数据集（真实定价-成交匹配数据）
- 竞品无法快速复制的壁垒
- 可以从"工具"变为"数据服务"
